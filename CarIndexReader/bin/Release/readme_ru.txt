### Назначение программы

Данная программа читает и редактирует структуру данных `FileStructure`, содержащую записи о машинах (CarRecord), а затем позволяет сохранить изменённые данные обратно в бинарный файл. Программа отображает поля структуры и даёт возможность изменять значения, в том числе выбирать варианты для полей enum из заранее подготовленных списков.

---

### Требования к окружению

1. **Файл `carindex.def`**  
   Должен находиться в одной папке с исполняемым файлом (EXE).  
   Размеры и формат данных должны соответствовать описанным в коде структурам.

2. **Файл `headers.txt`**  
   Должен находиться в одной папке с EXE.  
   Формат файла: каждая строка – это пара `ключ=значение`, где:
   - `ключ` соответствует имени поля структуры (например, `carNameId`, `enumA`, `zeroOrFour` и т.д.)
   - `значение` – человекочитаемое название или описание данного поля.  
   Эти заголовки будут отображаться на форме для удобства пользователя.

3. **Файл `enums.txt`**  
   Должен находиться в одной папке с EXE.  
   Формат файла:
   - Разделы Enum обозначаются `[имя_enum]`
   - Под каждым разделом идут строки `НазваниеПункта=Значение`
   - Значения могут быть в шестнадцатеричном формате `0x..` или в десятичном.

---

### Краткое описание логики работы

- При запуске приложение читает структуру из файла `carindex.def`.
- Загружает заголовки полей из `headers.txt` и варианты для enum из `enums.txt`.
- Отображает данные первого (по умолчанию `index = 0`) автомобиля (`CarRecord`) в форме.
- Пользователь может переключаться между машинами (81 записей) с помощью кнопок `Next Car` и `Previous Car`.
- Пользователь может изменять значения строковых полей, целочисленных полей и выбирать варианты из выпадающих списков для enum-полей.
- По нажатию `Save` изменения записываются обратно в структуру и сохраняются в исходный бинарный файл `carindex.def`.
